# STRESSED — Scope & Feature List

## Unique Selling Point (USP)

1. บรรยากาศ Lovecraftian Horror ที่สร้างจากสิ่งลึกลับที่ไม่อาจมองเห็นชัดเจน
2. ระบบสติและไฟฉายที่ให้ความรู้สึกกดดันและหลาบหลอน
3. โหมดสำรวจแบบเลือกจุดและแก้ปริศนาที่เหมาะกับเกม jam ที่มีเวลาจำกัด

## In Scope — สิ่งที่จะทำในเกมนี้

| # | Feature                                                          | Priority | หมายเหตุ                                                                                   |
| - | ---------------------------------------------------------------- | -------- | -------------------------------------------------------------------------------------------------- |
| 1 | ระบบแผนที่และเลือกพื้นที่               | Must     | ผู้เล่นเลือกจุดสำรวจจากแผนที่แทนการเดินอิสระ           |
| 2 | ระบบโต้ตอบกับจุดในฉาก                       | Must     | ซูมเข้าไปยังฉากย่อยเพื่อสำรวจและเก็บข้อมูล               |
| 3 | ระบบไฟฉายและความมืด                           | Must     | เพิ่มความรู้สึกกดดันและเปิดเผยสิ่งลึกลับในพื้นที่ |
| 4 | ระบบเก็บไอเทมและปลดล็อกเหตุการณ์ | Must     | เก็บไอเทมเพื่อไปสู่ปริศนาหรือพิธี                                 |
| 5 | ระบบสติ                                                   | Must     | ลด/เพิ่มสติจากเหตุการณ์และไอเทมต่าง ๆ                           |
| 6 | ปริศนาหรือมินิเกมอย่างน้อย 1 ชนิด  | Should   | ใช้ซ้ำในหลายจุดเพื่อประหยัดเวลา                                     |
| 7 | Ending อย่างน้อย 2 ชนิด                            | Should   | ชนะและแพ้แบบพื้นฐานเพียงพอสำหรับ jam                               |
| 8 | Art style เปลี่ยนไปตาม sanity                        | Must     | Art style  จะเปลี่ยนไปตาม stage ของค่า sanity                                 |

## Out of Scope — สิ่งที่จะไม่ทำ

- ระบบเดิน-วิ่งอิสระแบบเต็มพื้นที่
- ควบคุมตัวละครแบบ Action-heavy
- ภาคต่อหรือเนื้อเรื่องยาวหลายเงื่อนไข
- Art transition ที่ซับซ้อนมากเกินไป

## Risks & Assumptions

| ความเสี่ยง                                             | ผลกระทบ                             | แผนรับมือ                                                     |
| ---------------------------------------------------------------- | ------------------------------------------ | ---------------------------------------------------------------------- |
| เวลาในการพัฒนาไม่เพียงพอ                 | ระบบอาจไม่สมบูรณ์         | ลดจำนวนพื้นที่และปริศนาลงให้กระชับ   |
| เทคนิคภาพและแสงใน MonoGame ทำได้จำกัด | ความรู้สึกหลอนอาจลดลง | ใช้สี/แสง/เสียงเพื่อสร้างบรรยากาศแทน |
| ทีมยังไม่มีการแบ่งหน้าที่ชัดเจน   | การทำงานอาจล่าช้า         | จัดหน้าที่ตามความถนัดตั้งแต่วันแรก   |
